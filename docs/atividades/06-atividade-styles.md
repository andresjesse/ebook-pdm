# ⚡ Atividade Prática: Styles

Nesta atividade, vamos sair da teoria e entrar no campo da engenharia reversa visual. O desafio é transformar uma imagem estática de um aplicativo real em uma interface bem estruturada utilizando components e **Flexbox** no React Native.

## Objetivo

Durante a aula, escolhemos juntos um aplicativo de mercado (como Spotify, Instagram ou Netflix) e selecionamos uma tela para reproduzir. O professor iniciará o projeto, criando a estrutura básica do layout e hospedando o código em um repositório no GitHub.

Sua missão é dar continuidade a esse trabalho, refinando o design e adicionando os elementos que ainda faltam.

## Instruções da Atividade

### 1. **Clone o Projeto Base**

Faça um *clone* do repositório indicado pelo professor e configure o ambiente na sua máquina. 

O **código base** estará no branch `main` do seguinte repositório: [https://github.com/university-lessons/pdm-lesson-styles-layouts](https://github.com/university-lessons/pdm-lesson-styles-layouts)

Também faça o download do **print do aplicativo escolhido** (imagem base). Essa imagem será compartilhada no canal da disciplina no Discord.

### 2. **Análise Visual**

Observe atentamente o *print* da tela escolhida. Identifique as hierarquias visuais: o que é um container? O que é uma linha (`row`)? O que é uma coluna (`column`)?

### 3. **Desenvolvimento**

   - Você deve implementar pelo menos **3 novos elementos visuais** que ainda não foram codificados pelo professor.
   - Exemplos de elementos: Uma barra de pesquisa, um card de produto, um carrossel de categorias, um botão flutuante, etc.

### 4. **Uso de Flexbox**

Toda a estrutura de posicionamento deve ser feita utilizando as propriedades de Flexbox aprendidas em aula (evite posicionamentos absolutos e tamanhos fixos, a menos que seja estritamente necessário).

### 5. **Estilização**

Utilize o `StyleSheet` para organizar seus estilos de forma clara e legível.

## O que Você Deve Observar?

- **Fidelidade ao Design**: O quanto o seu código se aproxima visualmente do print original.
- **Domínio do Flexbox**: Uso correto de `justifyContent`, `alignItems`, `flexDirection` e `gap`.
- **Organização do Código**: Como você estruturou seus estilos e se manteve o padrão do projeto inicial.
- **Iniciativa**: A complexidade e o acabamento dos 3 elementos escolhidos para implementação.

---

:::tip Dica de Ouro
Antes de começar a escrever o código, tente "desenhar" os quadrados do Flexbox por cima da imagem original. Isso ajuda muito a visualizar como os `Views` devem ser aninhados.
:::
